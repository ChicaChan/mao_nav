# 博客标签规划（精简版）

## 目标
- 控制标签总量，避免同义词和事件名标签导致归档碎片化。
- 让读者在归档页能快速按主题检索，而不是看到大量一次性标签。

## 标准标签池（10 个）
- 技术类：`Kaggle`、`机器学习`、`特征工程`、`模型验证`、`模型集成`、`数据分析`
- 研究类：`市场研究`
- 评论类：`舆情分析`、`案例复盘`、`社会议题`

## 使用规则
1. 每篇文章使用 **2-4 个标签**，不超过 4 个。
2. 至少包含 1 个“主题标签”（如 `机器学习`、`舆情分析`）。
3. 优先使用通用标签，不再使用人名、具体事件名作为标签。
4. 工具名（如 XGBoost、LightGBM）写入正文关键词，不单独占用标签。

## 旧标签归并策略
| 旧标签 | 新标签 |
|---|---|
| CBC分析 / 联合分析 / 消费者偏好 / 定价策略 | 市场研究 |
| XGBoost / LightGBM / GPU加速 / 集成学习 / Stacking | 模型集成 |
| OOF / Out-of-Fold / 交叉验证 | 模型验证 |
| 舆情研判 / 公信力危机 / 视觉符号 / OSINT / 阶层对立 | 舆情分析 / 案例复盘 / 社会议题 |

## 执行与维护
- 新文章发布前，先对照本文件选标签。
- 若确需新增标签，先评估是否可被现有标签覆盖；只有可复用的长期主题才新增。
- 建议每月做一次标签巡检，清理新增的临时标签。
